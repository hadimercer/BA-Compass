1. Overview

This document presents two complementary artifacts that describe the user's journey through BA Compass from two perspectives: the system's behavioral logic (UML Activity Diagram) and the user's lived experience (User Journey Map).

Together, they serve as the bridge between the structural specifications in the FRD and Class Diagram and the human experience the product is designed to deliver.

3. User Journey Map

The user journey map captures the end-to-end experience of a junior-to-mid-level BA engaging with BA Compass — from first login through to downloading a completed artifact package. Unlike the Activity Diagram, this artifact documents the human experience layer: what the user is doing, thinking, feeling, and where the product creates or resolves friction.

Target user persona: A junior BA or recent graduate assigned to their first medium-complexity engagement. They are analytically capable but uncertain about methodology — what to do first, what belongs in each deliverable, and how to structure outputs that will pass scrutiny. They are working without a senior mentor.

3.3 Key Insight — The Emotional Arc

The emotional arc is the clearest indicator of where BA Compass creates value. Without the tool, a junior BA operates in a state of persistent low-grade anxiety: uncertain whether they're doing the right things in the right order, producing outputs that meet an invisible quality bar.

BA Compass restructures this experience into a predictable confidence curve. The peak dip at Phase 5 (gap analysis) is intentional and healthy — it represents the tool catching problems before they reach the stakeholder, which is precisely the professional value a senior BA mentor would provide. The final state at Phase 6 is high confidence rooted in completeness, not optimism.

3.4 Design Implications

Journey PhaseDesign ImplicationFRD Reference
Phase 1 — OnboardingEmpty dashboard must communicate purpose immediately — headline, CTA, and a short "how it works" summary. No blank whitespace.FR-04, NFR-07
Phase 2 — IntakeIntake UI must feel conversational, not form-like. Chat interface, not fields. AI reasoning for the classification must be visible.FR-08, FR-11, FR-12
Phase 3 — OrientationRoadmap view must be readable at a glance — visual progress indicators, module status colours, recommended-vs-optional distinction clear.FR-15, FR-17
Phase 4 — Deep WorkCo-pilot screen must show context (project dimensions, module purpose) without being cluttered. Refinement loop must be fluid.FR-19, FR-20, FR-21
Phase 5 — QualityGap report must be specific, not alarming. Frame gaps as "opportunities" not "failures." Traceability matrix should be scannable.FR-25, FR-26
Phase 6 — DeliveryExport screen must feel like a milestone moment. Clear scope selector, package contents summary, single prominent download CTA.FR-28, FR-29
