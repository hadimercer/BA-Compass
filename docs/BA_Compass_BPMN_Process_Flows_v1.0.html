1. Overview

This document presents four BPMN (Business Process Model and Notation) diagrams documenting the core user journeys through BA Compass. Each diagram uses standard BPMN notation with swimlanes to distinguish actor responsibilities.

All process flows are derived from the Functional Requirements Document (FRD v1.0) and the Use Case Diagram v1.0. Together they describe how the system behaves from the user's first interaction through to final deliverable export.

2. Diagram 1 — New Project Creation & Intake Classification

This is the most critical process flow. It covers the journey from project creation through conversational intake, AI classification, user confirmation, and live roadmap generation.

Process Description

1. User enters project name and description
Triggers project record creation in the database and opens the intake interface.
2. User describes their assignment in natural language
Free-text input (or pasted email/brief) is sent to the AI Engine for classification signal extraction.
3. AI extracts classification signals
The AI analyzes input for the four Must-Know dimensions. If insufficient signal exists, it asks follow-up questions (max 2 per turn). This loops until confidence is reached.
4. AI presents classification recommendation
Engagement Type and Scale Tier are recommended with plain-language explanations. Dimension values are persisted.
5. User accepts or overrides
Either path leads to roadmap generation from the corresponding template. Override uses the user-selected combination.
6. System generates live roadmap
Template modules are copied to project_roadmap, each with status Not Started. Process completes.

3. Diagram 2 — Roadmap Navigation & Module Selection

Covers the flow from opening a project through navigating the roadmap and selecting a module to work on, including dependency checking.

Process Description

1. User selects a project from the dashboard
The system loads the roadmap view (Navigator Mode) as the default landing page.
2. User browses the roadmap
All modules are visible with status indicators, knowledge area groupings, and completion percentage.
3. User clicks a module to enter
The AI checks whether the module's logical dependencies are complete.
4. Dependency check gateway
If dependencies are complete, the user enters co-pilot mode directly. If not, a warning is displayed identifying missing prerequisites. The warning is advisory only — the user can dismiss it and proceed.
5. Module status updates to In Progress
Automatic on first entry. Process completes with the user in co-pilot mode.

4. Diagram 3 — Co-Pilot Module Interaction

The core value delivery flow. Covers the guided conversation between the user and AI within a single module, from entry through artifact generation, editing, saving, and completion.

Process Description

1. Module loads with header and context
System displays module metadata (name, knowledge area, inputs, outputs) and loads any existing conversation history from the database.
2. AI initiates guided conversation
The AI asks targeted questions tailored to the module type, using all available project context to avoid redundancy.
3. Question-answer loop
The user answers questions. The AI processes responses, flags gaps (missing stakeholders, vague requirements), and checks if enough information exists to generate a draft. Each turn is persisted. The loop continues until the AI has sufficient input.
4. AI generates draft artifact content
A structured draft is generated and rendered in an editable view for the user.
5. User reviews and edits
The user can modify content directly. If unsatisfied, they can continue the conversation to refine. If satisfied, they save.
6. Save and completion
Artifact is saved as a new version. The AI provides a completion summary listing outputs produced and remaining gaps. The user can mark the module complete or continue working.

5. Diagram 4 — Artifact Export & Packaging

Covers both single artifact export and full project package export, including format selection and document generation.

Process Description

1. User initiates export
An exclusive gateway determines whether this is a single artifact export or a full project package export.
2a. Single artifact path
User selects the specific artifact and chooses an export format (Markdown, PDF, or Word).
2b. Package export path
User selects which artifacts to include in the package.
3. System retrieves artifact content from database
All selected artifact content is loaded from the artifacts table.
4. System generates formatted document(s)
Content is rendered into the chosen format with professional formatting.
5. Apply branding and cover page
Project name, engagement type, date, and artifact type are applied to headers/cover. Package exports include a summary cover page.
6. User downloads
The formatted export is downloaded to the user's device. Process completes.
